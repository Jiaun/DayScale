# 消费摊销 App 需求功能文档 v2

## 1. 项目定义

### 1.1 项目背景

用户买完手机、电脑、耳机、家电、会员等商品后，通常只能感知一次性支出，难以持续理解“这件东西平均每天相当于花了多少钱”。本项目希望把每件已购买商品转化为一个持续变化的日均指标，帮助用户更直观地复盘消费。

### 1.2 产品定位

这是一个面向个人的已购商品消费摊销工具。

- 我买的这件东西现在平均每天/每月/每年相当于多少钱
- 我所有东西合起来现在平均每天/每月/每年相当于多少钱
- 哪些东西当前“最贵”
- 我在哪些品类上花得最多

### 1.3 核心价值

- 把一次性消费转成长期可感知的数据
- 帮助用户判断购买是否划算
- 让用户看到全部已购物品的每日占用情况
- 为后续购买决策提供参考

## 2. 核心概念

### 2.1 业务对象

一条商品记录代表一件已经购买的物品或服务。

每条记录至少包含：

- 商品名称
- 购买价格
- 购买日期
- 商品分类
- 备注信息

### 2.2 核心指标

本产品围绕两个核心指标展开：

- 单个商品日均摊销
- 全部商品总日均摊销

### 2.3 示例说明

例如：

- 商品：手机
- 购买价格：`8000`
- 购买日期：`2025-01-01`

则：

- `2025-01-01`：`8000 / 1 = 8000 元/天`
- `2025-01-02`：`8000 / 1 = 8000 元/天`
- `2025-01-03`：`8000 / 2 = 4000 元/天`

说明：

- 本文档默认按你的示例口径定义需求
- 也就是购买日到次日这段时间仍按 `1` 天计算

## 3. 核心规则

### 3.1 默认计算规则

默认规则：

- 经过天数 = 购买日期到当前日期的日差，最小值为 `1`
- 单个商品日均摊销 = 商品价格 / 经过天数
- 总日均摊销 = 所有有效商品日均摊销之和

### 3.2 日期规则

- 只按日期计算，不考虑具体时分秒
- 购买日期不能晚于今天
- 同一天购买的商品允许重复录入

### 3.3 商品状态规则

商品状态分为：

- `正常`
  正常参与全部统计
- `删除`
  不再参与任何统计

### 3.4 金额显示规则

- 页面展示默认保留 `2` 位小数
- 排序按真实数值排序，不按格式化文本排序

### 3.5 边界规则

- 商品价格可以为 `0`，用于赠品等场景
- 商品价格不能小于 `0`
- 商品名称不能为空
- 超久以前购买的商品允许存在，其日均值会逐渐变小

## 4. 指标命名建议

### 4.1 命名问题

“日均消费”易懂，但可能让用户误以为是实际现金流。

更准确的表达建议：

- 首选：`日均摊销`

### 4.2 首页主指标建议

首页第一优先级建议显示：

- 当前总日均摊销


## 5. 第一版范围

### 5.1 第一版要做的功能

第一版聚焦于“录入商品 + 计算摊销 + 列表查看 + 基础统计”，具体包括：

- 新增商品
- 编辑商品
- 删除商品
- 首页总览
- 商品列表
- 商品详情
- 分类统计
- 基础搜索、排序、筛选

### 5.2 新增商品

用户可以新增一件商品，字段包括：

- 商品名称，必填
- 购买价格，必填
- 购买日期，必填
- 分类，选填
- 备注，选填
- 图片，选填

### 5.3 首页总览

首页需展示：

- 当前总日均摊销
- 商品总数
- 总原始消费金额
- 当前日均摊销最高商品
- 分类消耗占比最近新增
- 消费日均前3

首页展示形式要求：

- 以分区列表展示信息
- 不使用卡片式堆叠展示
- 各分区按信息优先级自上而下排列

### 5.5 商品列表

商品列表需支持：

- 按日均摊销排序
- 按购买时间排序
- 按价格排序
- 按分类筛选
- 按名称搜索

商品列表每行至少展示：

- 商品名称
- 分类
- 购买价格
- 购买日期
- 已持有天数
- 当前日均摊销

### 5.5 商品详情

商品详情页需展示：

- 商品基础信息
- 当前日均摊销
- 已持有天数
- 历史变化趋势
- 编辑入口
- 删除入口

详情展示形式要求：

- 以字段列表和分区列表展示
- 不使用卡片式信息块作为主要展示方式

### 5.6 编辑商品

支持编辑：

- 商品名称
- 购买价格
- 购买日期
- 分类
- 备注
- 图片

编辑后相关统计需同步更新。

### 5.8 删除商品

支持：

- 删除商品

### 5.9 分类统计

分类统计页需支持查看：

- 分类商品数
- 分类总原始消费
- 分类总日均摊销
- 分类下日均摊销最高商品

## 6. 页面结构

### 6.1 页面清单

MVP 建议页面如下：

1. 引导页
2. 首页
3. 商品列表页
4. 新增商品页/商品详情页 (可共用同一个页面)
5. 分类统计页
6. 我的 / 设置页
7. 新增/删除提醒页面

### 6.2 底部导航建议

- 首页
- 商品
- 新增
- 统计
- 我的
